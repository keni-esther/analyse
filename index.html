PRESENTATION: Etat des outils aujourd'hui(Chart.js, Plotly)

Chart.js

Chart.js reste une bibliothèque open source très populaire pour créer des graphiques interactifs dans les pages web.
Elle est simple à apprendre pour débuter, mais aussi flexible pour des projets plus avancés.
Version 4.x (actuelle) ajoute de nombreuses améliorations utiles pour les développeurs :
tree-shaking pour réduire la taille du bundle (tu charges seulement ce dont tu as besoin).
Palette de couleurs intégrée par défaut (facilite le styling sans config).
Performances accrues, même pour de gros jeux de données.
Cela veut dire que pour des applications web modernes (React, Vue, Svelte), Chart.js est un excellent choix pour intégrer des graphes dynamiques avec JavaScript

Nouvelles tendances et outils autour de Chart.js

Chartly – générateur d’images chart
Une API moderne qui prend une config Chart.js et génère des images chart PNG/SVG sur le cloud, sans serveur perso.
Utile pour :
Générer des graphiques serveur ou automatisés (emails, rapports PDF, API automatisée).
Intégration rapide avec backend ou microservices (pas besoin de navigateur pour rendre).
Pour les devs backend qui veulent externaliser le rendu Chart.js, c’est une optimisation intéressante.

Plotly

Support multi-langages
Plotly n’est pas juste Python :
il existe aussi Plotly.js pour JavaScript.
Il couvre beaucoup plus de types de graphiques que Chart.js (treemaps, Gantt, Sankey, etc.)

Nouvelles fonctionnalités en Plotly.js

Multi-axis shapes :
tu peux maintenant dessiner des formes qui traversent plusieurs subplots, très utile pour les dashboards complexes.
Événements hover/click élargis :
les interactions ne sont plus limitées aux points de données, tu peux écouter des clics sur toute la figure.
Cela montre que Plotly continue à devenir plus puissant pour les dashboards de données interactifs complexes, très utilisés dans l’industrie (finance, énergie, monitoring, etc.).

Plotly + Dash

Dash (le framework Python de Plotly) est régulièrement cité dans des offres d’emploi pour construire applications web analytiques avec dashboards. C’est très demandé pour construire des interfaces web interactives sans passer par JS pur.
